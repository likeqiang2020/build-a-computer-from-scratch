# SR 锁存器

## 先理解"反馈"

前面的加法器电路都是**组合逻辑**——输出只取决于当前输入，没有记忆。那怎么让电路"记住"东西？

关键就是**反馈（feedback）**：把输出接回输入。

### 用或门试一下

拿一个 74LS32 或门，把输出（3 号）接到输入（2 号），1 号引脚接按钮（默认低电平，按下变高）：

![反馈或门 Logisim 电路图](../images/03-feedback-or-gate-logisim.png)

**效果演示：**

LED 默认不亮（1 号低电平，输出 Q=0）：
![反馈或门 默认关](../images/03-feedback-or-gate-off.png)

按下按钮再松手，LED **还亮着**：
![反馈或门 按后保持亮](../images/03-feedback-or-gate-on.png)

**原理**：按下时 1 号高电平 → 输出 Q=1 → Q 反馈到 2 号 → 即使松手（1 号变回 0），2 号仍然为 1 → 输出保持 1。

> ⚠️ 注意：2 号引脚连在输出上，**不是悬空**，所以不适用 TTL"悬空默认高电平"的特性。它被输出管脚实实在在地驱动着。

这就是**反馈让电路"记住"了之前的状态**——最简单的记忆单元。

---

## SR 锁存器

**SR 锁存器（Set-Reset Latch）** 就是用反馈实现的**1 位记忆单元**，可以存储 0 或 1。它是寄存器、内存等所有存储电路的基础。

### 原理

SR 锁存器有两个输入和两个输出：

| 信号 | 名称 | 说明 |
|------|------|------|
| S    | Set（置位） | 将输出 Q 设为 1 |
| R    | Reset（复位） | 将输出 Q 设为 0 |
| Q    | 输出 | 存储的状态值 |
| Q̅   | 反相输出 | Q 的逻辑非 |

### 电路结构

两个**或非门（NOR）**交叉耦合：

- 上面 NOR：输入为 R 和下面门的输出 Q̅，输出为 Q
- 下面 NOR：输入为 S 和上面门的输出 Q，输出为 Q̅

### 工作逻辑

| S | R | Q 新状态 | 说明 |
|:-:|:-:|:--------:|:----|
| 0 | 0 | 保持 | **锁存**——维持当前值 |
| 1 | 0 | 1 | **置位**——设为 1 |
| 0 | 1 | 0 | **复位**——设为 0 |
| 1 | 1 | 无效 | **禁止**——Q=Q̅=0，破坏互补关系 |

### 真值表

| S | R | Q（新） | Q̅（新） | 状态 |
|:-:|:-:|:------:|:-------:|:----:|
| 0 | 0 | 不变 | 不变 | **保持** |
| 1 | 0 |   1    |    0    | **置位** |
| 0 | 1 |   0    |    1    | **复位** |
| 1 | 1 |   0    |    0    | **禁止** |

> 也可以用**与非门（NAND）**实现，此时 S 和 R 为低电平有效。

## 总结

- SR 锁存器是**时序逻辑电路**的起点——输出不仅取决于输入，还取决于之前的状态
- 组合多个锁存器可构成**寄存器**，存储多位数据
- 实际中常用改进的 **D 锁存器** 或 **JK 触发器**（避免禁止态）
