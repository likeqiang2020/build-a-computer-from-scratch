# SR 锁存器

## 先理解"反馈"

前面的加法器电路都是**组合逻辑**——输出只取决于当前输入，没有记忆。那怎么让电路"记住"东西？

关键就是**反馈（feedback）**：把输出接回输入。

### 用或门试一下

拿一个 74LS32 或门，把输出（3 号）接到输入（2 号），1 号引脚接按钮（默认低电平，按下变高）：

<img width="600" src="../images/03-memory/03-feedback-or-gate-logisim.png" alt="反馈或门 Logisim 电路图">

**效果演示：**

LED 默认不亮（1 号低电平，输出 Q=0）：  
<img width="600" src="../images/03-memory/03-feedback-or-gate-off.jpg" alt="反馈或门 默认关">

按下按钮再松手，LED **还亮着**：  
<img width="600" src="../images/03-memory/03-feedback-or-gate-on.jpg" alt="反馈或门 按后保持亮">

**原理**：按下时 1 号高电平 → 输出 Q=1 → Q 反馈到 2 号 → 即使松手（1 号变回 0），2 号仍然为 1 → 输出保持 1。

> ⚠️ 注意：2 号引脚连在输出上，**不是悬空**，所以不适用 TTL"悬空默认高电平"的特性。它被输出管脚实实在在地驱动着。

这就是**反馈让电路"记住"了之前的状态**——最简单的记忆单元。

## SR 锁存器

**SR 锁存器（Set-Reset Latch）** 就是用反馈实现的 **1 位记忆单元**。它是寄存器、内存等所有存储电路的基础。

### 电路结构

两个**或非门（NOR）**交叉耦合：

```
      R ──┤1     │
          │ NOR1 ├── Q
      Q̅ ──┤2     │

      S ──┤1     │
          │ NOR2 ├── Q̅
      Q  ─┤2     │
```

- NOR1 输入：R 和 Q̅，输出 Q
- NOR2 输入：S 和 Q，输出 Q̅

<img width="600" src="../images/03-memory/03-sr-latch-logisim.png" alt="SR 锁存器 Logisim 图">

### 锁存原理

"锁存"的关键在于 **S = R = 0** 时，反馈回路把输出值"锁"住了。我们来跟踪一下：

**假设当前 Q = 1（已置位）：**
```
       R=0 ──┤1     │
             │ NOR1 ├── Q=1  ← 因为 R=0、Q̅=0 → 0 NOR 0 = 1
        Q̅=0 ─┤2     │

       S=0 ──┤1     │
             │ NOR2 ├── Q̅=0  ← 因为 S=0、Q=1 → 0 NOR 1 = 0
        Q=1 ─┤2     │
```

**假设当前 Q = 0（已复位）：**
```
       R=0 ──┤1     │
             │ NOR1 ├── Q=0  ← 因为 R=0、Q̅=1 → 0 NOR 1 = 0
        Q̅=1 ─┤2     │

       S=0 ──┤1     │
             │ NOR2 ├── Q̅=1  ← 因为 S=0、Q=0 → 0 NOR 0 = 1
        Q=0 ─┤2     │
```

两种状态都**稳定自持**：每个或非门的输出都反过来维持对方的输入，形成一个闭环。这就是"锁"的实质——**状态被反馈回路固定住了**，外部不给信号就不会变。

> 可以理解为两个 or 非门互相"顶"着对方：一个输出 1 就把另一个按住输出 0，按住的这个又回头把对方稳住。

### 工作逻辑

| S | R | Q（新） | Q̅（新） | 说明 |
|:-:|:-:|:------:|:-------:|:----|
| 0 | 0 | **保持** | **保持** | **🔒 锁存**——输出维持之前的值不变 |
| 1 | 0 |   1    |    0    | **置位**（Set）——写入 1 |
| 0 | 1 |   0    |    1    | **复位**（Reset）——写入 0 |
| 1 | 1 |   0    |    0    | **禁止**——破坏互补关系，应避免 |

> **重点**：只有 S 或 R 为 1 时才会改变状态；当两者都回到 0，电路**锁住当前值不丢**。这就是"锁存器"名字的由来。

> 也可以用**与非门（NAND）**实现，此时 S 和 R 为低电平有效。

### 实物搭建

用 **74LS02**（四路 2 输入或非门）来搭：

![74LS02 或非门](../74-series/74LS02-quad-nor.png)

接线（左右对称）：

```
电源+5V ──┬─ 开关L ── 2号引脚（左侧或非门输入1）
          │
          └─ 开关R ── 5号引脚（右侧或非门输入1）

左侧或非门：输入 2号、3号 → 输出 1号
  ├─ 3号 ← 4号（右侧输出，即 Q̅）
  ├─ 1号 → LED_L（经220Ω接地）
  └─ 1号 → 6号（右侧输入2）

右侧或非门：输入 5号、6号 → 输出 4号
  ├─ 6号 ← 1号（左侧输出，即 Q）
  ├─ 4号 → LED_R（经220Ω接地）
  └─ 4号 → 3号（左侧输入2）
```

### 实测演示

**1. 上电瞬间**：两个输入都是 0 → 输出都是 1 → 互相反馈产生竞争，随机一个 LED 亮

<img width="600" src="../images/03-memory/03-sr-latch-power-on.jpg" alt="上电初始">

**2. 左侧开关置 1（Set）** → 左侧灭、右侧亮（写入 1）

<img width="600" src="../images/03-memory/03-sr-latch-set-1-on.jpg" alt="Set 按下">

**3. 左侧开关回到 0** → S=0、R=0，**锁存生效**，右侧仍亮（记住了一直是置位状态）

<img width="600" src="../images/03-memory/03-sr-latch-set-1-off.jpg" alt="Set 松手后保持">

**4. 右侧开关置 1（Reset）** → 右侧灭、左侧亮（写入 0）

<img width="600" src="../images/03-memory/03-sr-latch-reset-1-on.jpg" alt="Reset 按下">

**5. 右侧开关回到 0** → S=0、R=0，**再次锁存**，左侧仍亮（记住了一直是复位状态）

<img width="600" src="../images/03-memory/03-sr-latch-reset-1-off.jpg" alt="Reset 松手后保持">

**6. 两侧开关都置 1（禁止态）** → 两个 LED 都灭

<img width="600" src="../images/03-memory/03-sr-latch-both-1-off.jpg" alt="禁止态">

> 💡 LED 记得串 **220Ω 电阻**再接地，不然红色 LED 压降低，可能拉低输出电平导致反馈失效。

## 总结

- **锁存器的核心**：S=0、R=0 时，反馈回路把输出值牢牢"锁"住，外部不给新信号就不会变
- SR 锁存器是**时序逻辑电路**的起点——输出不仅取决于输入，还取决于之前的状态（记忆）
- 组合多个锁存器可构成**寄存器**，存储多位数据
- 实际中常用改进的 **D 锁存器** 或 **JK 触发器**（避免禁止态）
