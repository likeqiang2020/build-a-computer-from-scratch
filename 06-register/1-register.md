# 寄存器

上个章节说到了总线，多个 D 触发器的输入和输出接入到总线的话，会出现两个问题：
1. 如何控制某一个 D 触发器**读取**总线数据？
2. 如何控制某一个 D 触发器**输出**数据到总线？

## 解决问题 1：Load 控制

比如三个 D 触发器接入到总线上，现在只想让触发器 A 读取总线数据，触发器 B、C 不读取。可以给每个触发器再增加一个引脚——**Load**，只有 Load=1 时，触发器才能从总线上读取数据。

## 解决问题 2：Enable 控制

三个 D 触发器接入到总线上，现在只想让触发器 A 输出到总线上。增加一个引脚——**Enable**，只有 Enable=1 时，触发器才能把数据输出到总线上。

## 寄存器

增加 Load 和 Enable 的 D 触发器，我们称之为**寄存器**：

```
数据D─       ┌───────┐
时钟Clk─     │ 寄存器 │─── 输出 Q
加载Load─    │       │
使能Enable─  └───────┘
```

### Load 设计思路

当 Load=1 时，数据 D 写入到寄存器里；Load=0 时，寄存器的输出保持不变。用组合逻辑来实现：

```
Load ──非门── 与门A输入1
 │              │
 │      ┌──与门A输出───┐
 │      │              │
 │      │   或门输入1───┤
 │      │              │─── 或门输出 ── D触发器的D
 │      └──或门输入2───┘
 │──与门B输入1          │
      与门B输出─────────┘
数据D ──与门B输入2
```

![Load 控制电路 Logisim 图](../images/06-register/register-logisim-load.png)

### Enable 设计思路

Enable 和输出 Q 接入一个**三态门**，用 Enable 控制 Q 是否输出到总线上。因为 Tinkercad 和 Logisim 里没找到三态门，就不画图了。

## 实际芯片：74LS173

实际上，**74LS173** 就是一个 4 位寄存器：

引脚图：

![74LS173 引脚图](../images/06-register/74ls173-pinout.png)

内部结构图：

![74LS173 内部结构](../images/06-register/74ls173-internal.png)

| 引脚 | 功能 |
|------|------|
| 1D~4D | 数据输入 |
| 1Q~4Q | 数据输出 |
| M、N | **Output Control**（即 Enable）— 接 VCC 不输出，接 GND 输出 |
| G1、G2 | **Data Enable**（即 Load）— 接 GND 可读取，接 VCC 不读取 |
| CLK (pin7) | 时钟输入（上升沿触发） |

## 面包板实测

图片中：
- 上面的面包板：左侧是 555 定时器做的自动时钟，右侧是 SR Latch
- 中间的面包板：每列有 1 个 220Ω 电阻，上接 GND，下接来自总线的 LED，用来展示当前总线的值
- 下面的两个面包板：左侧是 4 位 DIP 开关（输出到总线，方便手动设置总线值）；中间是总线；右边是 74LS173 构成的 4 位寄存器

将 DIP 拨为 1001，4 位寄存器的 CLK 接 SR 的 Q，4 个输入接总线，4 个输出接 LED 再接 GND，Enable 和 Load 都接 GND（允许读写）。刚上电时，总线 LED 同步显示 1001，寄存器输出为随机值：

![上电初始状态 总线 1001 寄存器随机](../images/06-register/register-before-clock.png)

按下 SR Latch 的 R、S 键，模拟 1 个时钟信号，寄存器同步将总线值读取并输出，LED 变为 1001：

![时钟触发后 寄存器输出 1001](../images/06-register/register-after-clock.png)

> 总线是 8 位的，但本次只用了 4 位，接下来将扩展寄存器和总线到 8 位。
